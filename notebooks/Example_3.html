Tutorial : Document and Page methods

This tutorial shows examples on how to use some methods in the Document and Page class. All methods in the Document class are also available at the Page level. Page is an entity of the Document and is.

Note: This is a highly simplified tutorial and functions in this package can be used without simplifying steps each into a line of code.

Imports

After installation, Textractor is to be imported.

[1]:

import os
from PIL import Image
from textractor import Textractor
from textractor.visualizers.entitylist import EntityList
from textractor.data.constants import TextractFeatures, Direction, DirectionalFinderType

Initialize Textractor

To initialize the Textractor, the customer can pass the profile-name stored in their ~/.aws/config file to autoload credentials. These are the credentials to internally initialize the boto3 client. Additional parameters can be passed to the function to store output on the S3 bucket.

[2]:

extractor = Textractor(aws_profile_name='default')

Input Document

Here’s a look at the document we wish to run Textractor on.

[3]:

Image.open(os.path.join(os.getcwd(), 'tests/fixtures/acord_form_2_05272022.png'))

[3]:

Call Textract

To process an image into a single document object, we need to call an sync API. All the Textractor API endpoints support local filepaths, S3 filepaths or list of PIL Images being passed to the file_source parameter. For this example, we pass an S3 location. Additionally, we want a clear categorization of the document entities so we call the analyze_document() function with the necessary TextractFeatures passed to the features parameter.

[4]:

document = extractor.analyze_document(file_source='s3://bucket-name/acord_form_2_05272022.png',
                                        features=[TextractFeatures.TABLES, TextractFeatures.FORMS],
                                        s3_prefix='', save_image=True)

Task 1 : Numpy Indexing on Tables

Customer might want to access a table and edit its structure for their use-case.

Steps to perform task

To accomplish this task, the user would simply have to:

  1. Access tables from the document object.

  2. Choose the table they wish to manipulate.

  3. Display the results of manipulation to verify format and contents.

  4. Beyond this, the customer can further manipulate the table or choose to export it.

Note: This is a highly simplified tutorial and function in this package can be used without simplifying steps each into a line of code.

Retrieval of all tables in the document

Table data is stored at the document and page level as a property and can be accessed as shown below.

[5]:

# All tables present in the document
tables = document.tables

Choose desired table among all the document tables

[6]:

# Choose specific table
chosen_table = tables[1]

Filter and display table

To make the interface of accessing 2D data intuitive, the table filtering supports numpy indexing. Finally, to print the table contents in a structured manner, the pretty_print() method from EntityList class can be called upon it.

[8]:

filtered_table = chosen_table[:, 2::]
table_string = EntityList(filtered_table).pretty_print()

Task 2: Retrieve key-values and checkboxes by geometry using directional_finder()

It is possible to retrieve key-values and checkboxes present in a certain location of the document using reference words and direction of the required key-values and checkboxes with respect to them.

Steps to perform task

  1. First we select the page we want to perform these queries on.

  2. We identify the headers between which we want to extract these key-values or checkboxes.

  3. Prefix is optional. By not passing the paramater a value, we retrieve the entities unaltered.

  4. We visualize the results.

[9]:

page = document.page(0)

[10]:

page.directional_finder( word_1="employee benefits and liability", word_2="primary location and subsidiaries", prefix ="",
               direction=Direction.BELOW, entities=[DirectionalFinderType.KEY_VALUE_SET], edit_original=False)
