Export Form Data

Customer would like to export all key-values extracted from a list of images as a .csv file.

Steps to perform task

To accomplish this requirement, the user would have to perform the following tasks.

  1. Call Textract Async API StartDocumentAnalysis.

  2. Parse the response JSON.

  3. Find a way to identify KEY_VALUE_SET blocks that are related as Keys and Values.

  4. Export in csv format

Solution with Textractor

Imports

After installation, Textractor is to be imported.

[1]:

import os
from PIL import Image
from textractor import Textractor
from textractor.data.constants import TextractFeatures

Initialize Textractor

To initialize the Textractor, the customer can pass the profile-name stored in their ~/.aws/config file to autoload credentials. These are the credentials to internally initialize the boto3 client. Additional parameters can be passed to the function to store output on the S3 bucket.

[2]:

extractor = Textractor(aws_profile_name='default')

Call Textract

To process multiple images into a single document object, we need to call an async API. All the Textractor API endpoints support local filepaths, S3 filepaths or list of PIL Images being passed to the file_source parameter. Additionally, we want a clear categorization of the document entities so we call the start_document_analysis() function with the necessary TextractFeatures passed to the features parameter.

[3]:

image_1 = Image.open(os.path.join(os.getcwd(), 'tests/fixtures/invoice1.png'))
image_2 = Image.open(os.path.join(os.getcwd(), 'tests/fixtures/invoice2.png'))
document = extractor.start_document_analysis(file_source=[image_1, image_2],
                                               features=[TextractFeatures.TABLES, TextractFeatures.FORMS],
                                               s3_prefix='',
                                               client_request_token='',
                                               job_tag='',
                                               save_image=True)
